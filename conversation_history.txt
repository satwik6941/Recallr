### Overview
An agentic RAG system can fetch information from a set of documents and generate contextually relevant responses. In this implementation, we'll use **LlamaIndex** to index documents and **OpenAI** models for both retrieval and generation.

### Code Example

Below is a comprehensive Python code snippet to set up your agentic RAG system:

```python
import os
from typing import List
from llama_index.core import (
    VectorStoreIndex,
    SimpleDirectoryReader,
    StorageContext,
    load_index_from_storage,
    Settings
)
from llama_index.core.node_parser import SentenceSplitter
from llama_index.core.tools import FunctionTool, QueryEngineTool
from llama_index.core.agent import ReActAgent
from llama_index.llms.openai import OpenAI
from llama_index.embeddings.openai import OpenAIEmbedding
from dotenv import load_dotenv

# Load environment variables
load_dotenv()
os.environ["OPENAI_API_KEY"] = os.getenv("OPENAI_API_KEY")

# Set up LlamaIndex settings
Settings.llm = OpenAI(model="gpt-4o-mini", temperature=0.1)
Settings.embed_model = OpenAIEmbedding(model="text-embedding-3-small")

class AgenticRAG:
    def __init__(self, documents_path: str = None, index_path: str = "./storage"):
        self.documents_path = documents_path
        self.index_path = index_path
        self.index = None
        self.agent = None

    def load_documents(self) -> List:
        if not self.documents_path:
            raise ValueError("Documents path not provided")

        print(f"Loading documents from {self.documents_path}")
        documents = SimpleDirectoryReader(self.documents_path).load_data()
        print(f"Loaded {len(documents)} documents")
        return documents

    def create_index(self, documents: List) -> VectorStoreIndex:
        print("Creating index...")
        node_parser = SentenceSplitter(chunk_size=1024, chunk_overlap=200)
        nodes = node_parser.get_nodes_from_documents(documents)

        self.index = VectorStoreIndex(nodes)
        print("Index created successfully")
        self.index.storage_context.persist(persist_dir=self.index_path)
        print(f"Index persisted to {self.index_path}")
        return self.index

    def load_index(self) -> VectorStoreIndex:
        print(f"Loading index from {self.index_path}")
        storage_context = StorageContext.from_defaults(persist_dir=self.index_path)
        self.index = load_index_from_storage(storage_context)
        print("Index loaded successfully")
        return self.index

    def setup_agent(self) -> ReActAgent:
        if not self.index:
            raise ValueError("Index not created or loaded")
        
        print("Setting up agent...")
        query_engine = self.index.as_query_engine(similarity_top_k=3)

        query_engine_tool = QueryEngineTool.from_defaults(
            query_engine=query_engine,
            name="document_query_engine",
            description="Useful for answering questions about the documents in the index"
        )

        def multiply(a: float, b: float) -> float:
            return a * b

        multiply_tool = FunctionTool.from_defaults(
            fn=multiply,
            name="multiply_numbers",
            description="Useful for multiplying two numbers"
        )

        self.agent = ReActAgent.from_tools(
            [query_engine_tool, multiply_tool],
            llm=Settings.llm,
            verbose=True
        )

        print("Agent setup complete")
        return self.agent

    def query(self, query_str: str) -> str:
        if not self.agent:
            raise ValueError("Agent not set up")
        
        print(f"Querying agent with: {query_str}")
        response = self.agent.chat(query_str)
        return str(response)

def main():
    documents_path = "./data"  # Path to your documents
    index_path = "./storage"  # Where to store the index

    rag_system = AgenticRAG(documents_path=documents_path, index_path=index_path)

    try:
        rag_system.load_index()
    except:
        documents = rag_system.load_documents()
        rag_system.create_index(documents)

    rag_system.setup_agent()

    queries = [
        "What is the main topic of the documents?",
        "Summarize the key points in the documents",
        "What is 5 multiplied by 7?",
        "Combine the information from the documents with the multiplication result"
    ]

    for query in queries:
        print(f"\nQuery: {query}")
        response = rag_system.query(query)
        print(f"Response: {response}")

if __name__ == "__main__":
    main()
```

### Explanation of the Code

1. **Initialization**: The `AgenticRAG` class initializes with paths for your documents and the index storage location.
   
2. **Loading Documents**: The `load_documents` method imports documents from a specified directory.
   
3. **Creating the Index**: The `create_index` method generates a vector index from the loaded documents, allowing for efficient retrieval.
   
4. **Index Loading**: The `load_index` method helps in fetching an existing index from storage.

5. **Setting Up the Agent**: The `setup_agent` method prepares the ReAct agent with necessary tools for querying documents and performing calculations.

6. **Querying the Agent**: The `query` method sends questions to the agent and retrieves the corresponding responses.

7. **Main Execution**: The `main` function orchestrates the overall flow, nicely showing how to load documents, create/load the index, and query the agent.

**What is Hydraulics?**
Think of hydraulics as using water (or fluids) to help machines move or lift heavy things, just like how our muscles help us lift. For example, if you have a balloon filled with water and you squeeze it, the water pushes out the other end. That's like how hydraulic systems work!

### Types of Hydraulics

1. **Hydraulic Pumps**: Imagine you have a toy water fountain! The pump is the part that pulls water up from a tank and pushes it out. There are two main types:
   - **Gear Pumps**: These work like two gears that fit together and spin. When they turn, they push the water out just like cranking a toy car.
   - **Piston Pumps**: Think of these like a large syringe. You pull the plunger to fill it with water, and when you push it down, the water shoots out. These can be used in big machines to lift heavy things.

2. **Hydraulic Motors**: These are the engines that use water to create movement. Think of them as the "muscles" of a machine!
   - **Gear Motors**: They work like the gear pumps we mentioned earlier, turning the water's power into movement, like pedaling a bike.
   - **Piston Motors**: These work like the piston pumps but do the opposite—taking pressure from water to turn and spin, similar to how wind turns a windmill.

### Examples of Hydraulics
- **Excavators**: Think of a giant robot digging into the ground. The hydraulic pumps in the excavator help it lift dirt and move its arms up and down. 
- **Brakes in Cars**: When you step on the brake pedal, you’re using hydraulics! Hydraulic fluid moves through pipes, helping the car stop safely.

Just like with the agentic RAG system we discussed earlier, which uses complex components to generate relevant information, hydraulic systems use various parts to work together to perform specific tasks effectively.
